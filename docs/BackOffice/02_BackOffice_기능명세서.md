## 1. 문서 개요

### 1.1 목적
본 문서는 맑은 message 운영을 위한 BackOffice 시스템의 화면별·기능별 동작을 정의한다.
각 메뉴의 입력/출력, 비즈니스 규칙, 외부 연동, 권한, 예외 처리를 명시하여 개발/QA의 기준 문서로 사용한다.

### 1.2 표기 규칙
- 권한: SUPER / OPS / REVIEWER / FINANCE / CS / SYSTEM / READONLY
- 상태값은 화면 라벨과 코드값을 함께 표기 (예: 승인 `APPROVED`)
- 모든 변경 작업은 감사 로그(Audit Log)에 기록한다.
- 모든 목록 화면은 공통적으로 검색·정렬·페이지네이션·엑셀 다운로드를 지원한다.

### 1.3 공통 비기능 요구
| 항목 | 정책 |
|------|------|
| 인증 | 운영자 ID + 비밀번호 + 2FA(OTP/이메일) 필수 |
| 세션 | 30분 무활동 시 자동 로그아웃 |
| 비밀번호 정책 | 10자 이상, 영문/숫자/특수문자 3종 조합, 90일 주기 변경 |
| 접근 제어 | 사내망 IP 화이트리스트, RBAC |
| 감사 로그 | 모든 변경/검수/환불/단가/권한 변경 이력 보관 (최소 5년) |
| 개인정보 | 화면 표시 시 휴대폰/이메일 마스킹, 다운로드 시 사유 입력 |
| 다국어 | 한국어 (영어는 차후) |

## 2. 대시보드

### 2.1 운영 현황 대시보드 `/admin/dashboard/operation`
**권한**: ALL
**기능**
- 실시간 발송 KPI 카드 (오늘 발송 건수 / 처리율 / 실패율 / 지연건수)
- 채널별 라이브 차트 (SMS, 알림톡, RCS, 이메일, PUSH) — 1분 단위 갱신
- 최근 5분간 큐 적체 추이
- 알림(Alert) 영역: 외부 연동 장애, 큐 임계치 초과
**비즈니스 규칙**
- 큐 길이가 임계치(기본 10,000)을 초과하면 빨간 배지 표출 + 시스템관리자에게 알림
- 채널 실패율 5% 초과 시 노란 배지 표출
**입력**: 기간 필터(오늘/어제/7일/30일), 채널 필터
**출력**: 카드 KPI, 시계열 차트, 이벤트 리스트

### 2.2 매출/크레딧 현황 `/admin/dashboard/revenue`
**권한**: FINANCE, OPS
**기능**
- 일·주·월·누적 매출 카드
- 결제 수단별/채널별 매출 비중
- 미결제 / 결제 실패 누적 건수
- 전체 고객사 잔여 크레딧 합계
**비즈니스 규칙**: 매출은 PG 승인 기준, VAT 제외 표기를 기본값으로 한다(설정에서 포함 가능).

### 2.3 회원/고객사 현황 `/admin/dashboard/members`
**권한**: OPS
**기능**: 신규 가입, 활성, 휴면 전환, 탈퇴 추이 / 고객사 증감

### 2.4 시스템 상태 `/admin/dashboard/system`
**권한**: SYSTEM
**기능**: 외부 연동 헬스체크 (PG / 문자사 / 카카오 / 통신사 / FCM / APNs / SMTP / S3), 큐/워커 상태

## 3. 회원/고객사 관리

### 3.1 고객사 목록 `/admin/members/companies/list`
**권한**: OPS
**검색 필터**: 고객사명, 사업자번호, 담당자, 가입일, 상태(활성/정지/탈퇴), 플랜
**리스트 컬럼**: 고객사ID, 사업자번호, 고객사명, 대표자, 가입일, 플랜, 보유 크레딧, 상태, 액션(상세/정지/탈퇴)
**기능**: 검색, 정렬, 엑셀 다운로드, 일괄 상태 변경(정지/해제), 신규 등록 버튼
**규칙**: 일괄 상태 변경 시 사유 필수 입력, 변경 즉시 해당 고객사의 모든 회원 토큰 무효화

### 3.2 고객사 상세/등록 `/admin/members/companies/detail`
**권한**: OPS
**탭 구성**
1. 기본 정보 — 고객사명/사업자번호/대표자/주소/플랜/계약기간/세금계산서 이메일
2. 담당자 — 담당자 추가/변경 (이름/직책/이메일/연락처)
3. 발신정보 — 등록된 발신번호/도메인/브랜드/프로필/PUSH 인증서 요약
4. 발송 통계 — 채널별/기간별 발송량 그래프
5. 결제 — 충전/사용 내역, 단가 오버라이드(요금정책 링크)
6. 메모 — 운영자 메모 (시간순 기록, 작성자 표시)
**규칙**: 사업자번호 중복 등록 금지, 변경 시 변경 이력 자동 기록

### 3.3 회원 목록 `/admin/members/users/list`
**권한**: OPS
**검색 필터**: 회원ID, 이메일, 이름, 휴대폰, 소속 고객사, 상태, 가입일, 마지막 로그인
**리스트 컬럼**: 회원ID, 이메일(마스킹), 이름, 소속 고객사, 권한, 마지막 로그인, 상태, 액션
**기능**: 회원 강제 로그아웃, 비밀번호 초기화, 정지/해제, 탈퇴 처리, 엑셀 다운로드(사유 입력)

### 3.4 회원 상세 `/admin/members/users/detail`
**권한**: OPS
**탭**: 기본 정보 / 권한 / 로그인 이력 / 발송 이력 / 메모
**기능**
- 권한 그룹 변경 (변경 시 사유 필수, Audit Log 기록)
- 보안 설정: 2FA 강제 사용, IP 제한
- 휴면 해제

### 3.5 권한 그룹 관리 `/admin/members/roles`
**권한**: SUPER, SYSTEM
**기능**
- 권한 그룹(역할) CRUD
- 메뉴×액션(조회/등록/수정/삭제/승인/다운로드) 매트릭스 편집
- 그룹 삭제는 소속 회원 0명일 때만 가능
**규칙**: SUPER 그룹은 삭제 및 권한 축소 불가

### 3.6 로그인 이력 `/admin/members/login-logs`
**권한**: OPS, SYSTEM
**기능**: 로그인 성공/실패, IP, 디바이스, User-Agent, 사유 표시 / 5회 연속 실패 시 자동 잠금 표시 / 운영자 잠금 해제

### 3.7 탈퇴 회원 관리 `/admin/members/withdrawn`
**권한**: OPS
**기능**: 탈퇴 사유, 보존 기간(개인정보처리방침에 따른 30~180일) 만료 임박 표시, 만료된 회원 데이터 익명화 처리

## 4. 발송 운영 모니터링

### 4.1 실시간 발송 현황 `/admin/sending/monitor/realtime`
**권한**: OPS
**기능**
- 채널별 TPS 게이지, 1초/5초/1분 평균
- 워커별 처리 건수
- 타임라인 (최근 1시간)
**규칙**: 실시간 데이터는 WebSocket/SSE로 업데이트, 끊김 시 자동 재연결 + Toast 알림

### 4.2 발송 큐/처리율 `/admin/sending/monitor/queue`
**권한**: OPS
**기능**: 채널별 큐 길이/소화율/지연 누적 건수, 워커 일시정지/재개 (SUPER 권한)

### 4.3 도달률/실패율 `/admin/sending/monitor/delivery`
**권한**: OPS
**기능**: 채널별·통신사별·고객사별 도달률·실패율, 실패 사유 Top 10

### 4.4 SMS 발송 이력 `/admin/sending/history/sms`
**권한**: OPS
**검색 필터**: 발송일, 고객사, 발신번호, 수신번호, 상태(대기/발송중/성공/실패/취소), 메시지 종류(SMS/LMS/MMS)
**컬럼**: 메시지ID, 고객사, 발송시각, 발신/수신, 종류, 상태, 사유, 단가, 미리보기
**기능**: 본문 마스킹 표시 + “원문 보기” 시 사유 입력, 단건 재발송, 상태 강제 종료(SUPER), 엑셀 다운로드(비동기)

### 4.5 알림톡 발송 이력 `/admin/sending/history/kakao`
**권한**: OPS
**기능**: SMS와 동일 + 템플릿코드, 대체발송 사용 여부, 카카오 응답 코드 표시 / 클릭 이벤트 통계

### 4.6 RCS / 4.7 이메일 / 4.8 PUSH 발송 이력
**권한**: OPS
**기능**:
- RCS: 브랜드/포맷(SMS/LMS/MMS Cell/Standalone), 통신사 응답 코드
- 이메일: 발송/오픈/클릭/바운스/스팸 신고 표시, 메시지 헤더 보기
- PUSH: 디바이스 토큰 단위 결과(성공/실패/만료), iOS/Android 분리 통계

### 4.9 캠페인 모니터링 `/admin/sending/campaigns`
**권한**: OPS
**기능**: 진행중/예약/완료 캠페인, 진척률, 채널별 발송수, 강제 일시중지/중단(SUPER), 운영 메모

### 4.10 플로우(복합) 모니터링 `/admin/sending/flows`
**권한**: OPS
**기능**: 노드별 진척률, 분기 비율, 노드 단위 실패 건수

### 4.11 실패/재처리 관리 `/admin/sending/retry`
**권한**: OPS
**기능**
- 실패 사유별 그룹화, 채널별/고객사별 필터
- 일괄 재발송 (최대 N건 제한, 재발송 시 단가 재계산)
- 재발송 사유 입력 필수, Audit Log 기록

### 4.12 차단/스팸 메시지 관리 `/admin/sending/spam`
**권한**: OPS
**키워드 필터** `/admin/sending/spam/keywords`
- 금칙어 등록(완전일치/부분일치/정규식), 대상 채널, 처리(차단/경고/검수)
- 우선순위 정렬, 활성/비활성 토글, 변경 이력
**스팸 신고 처리** `/admin/sending/spam/reports`
- 통신사·KISA 신고 접수 → 발신 고객사 통보 → 발신번호 일시정지 워크플로우
- 처리 SLA(예: 24시간 내) 표시 및 미처리 건 알림

## 5. 발신 정보 검수

### 5.1 발신번호 검수 `/admin/sender/numbers`
**권한**: REVIEWER
**상태**: 접수 `RECEIVED` / 검토중 `IN_REVIEW` / 승인 `APPROVED` / 반려 `REJECTED` / 보완요청 `REVISION`
**기능**
- 신청 목록(검색·필터·SLA 잔여시간), 신청 상세
- 첨부 서류 뷰어 (사업자등록증, 통신서비스 가입증명원 등) — 워터마크
- 승인/반려/보완요청 시 사유 필수, 사용자에게 메일/SMS 알림 자동 발송
**규칙**
- 동일 번호 중복 등록 검증
- 반려 후 동일 사유로 3회 반복 시 운영자 알림
- 승인 시 KISA 발신번호 등록 시스템과 연동

### 5.2 도메인 검수 `/admin/sender/domains`
**권한**: REVIEWER
**기능**: 도메인 소유 인증(TXT 레코드), DKIM 셀렉터 발급/검증, SPF 레코드 가이드, DMARC 권장 설정 안내

### 5.3 브랜드(RCS) 검수 `/admin/sender/brands`
**권한**: REVIEWER
**기능**: 브랜드 정보(로고/색상/대표번호), 통신사 등록 요청 발행, 통신사 응답 동기화

### 5.4 발신 프로필(카카오) 검수 `/admin/sender/profiles`
**권한**: REVIEWER
**기능**: 카카오 비즈채널/검색용 ID 매핑, 토큰 발송, 그룹(프로필 묶음) 관리

### 5.5 PUSH 인증서 검증 `/admin/sender/push-cert`
**권한**: REVIEWER
**기능**
- iOS APNs 인증서(.p8/.p12) 등록 시 만료일 자동 파싱
- Android FCM 서비스계정 키 검증 (테스트 발송 1건 자동 수행)
- 만료 30/14/7일 전 알림

### 5.6 080 수신거부번호 신청 처리 `/admin/sender/optout-080`
**권한**: REVIEWER
**기능**: 신청 → 통신사 회선 발급 → 사용자 매핑, 신청 취소/회선 회수 처리

## 7. 결제/크레딧 관리

### 7.1 충전 내역 `/admin/billing/charges`
**권한**: FINANCE
**검색 필터**: 기간, 고객사, 결제수단, 상태(승인/취소/부분취소/실패), 금액 범위
**기능**: 충전 상세, 영수증 보기, 결제수단 마스킹, 엑셀 다운로드(사유 입력)

### 7.2 결제 내역(PG) `/admin/billing/payments`

## 15. 공통 유스케이스/시퀀스

### 15.1 발신번호 검수 시퀀스
1. 사용자 → 발신번호 등록 신청 (서류 첨부)
2. 시스템 → 검수자 그룹에 알림
3. 검수자 → 신청 상세 검토 / 보완요청·반려·승인
4. 승인 시 → KISA 발신번호 등록 시스템 호출 → 결과 동기화
5. 사용자 알림(메일/SMS) + Audit Log

### 15.2 환불 처리 시퀀스
1. 사용자 → 환불 신청 (사유, 첨부)
2. 회계 → 정책에 따른 환불 가능액 자동 계산 표시
3. 회계 → 승인/반려, SUPER 결재
4. 승인 시 → PG 부분취소/취소 → 크레딧 차감 → 영수증 마이너스 발행
5. 사용자 알림 + Audit Log

### 15.3 점검 모드 적용 시퀀스
1. SYSTEM → 점검 시작/종료 시각 입력 + 사전 공지 게시
2. 시작 시각 도달 → 일반 사이트 라우팅을 점검 페이지로 전환
3. 종료 시 자동 해제 + 운영자 알림

### 15.4 단가 변경 시퀀스
1. FINANCE → 채널별 단가 변경 입력 + 적용 시작일 + 사유
2. SUPER 결재
3. 적용 시작일에 도달 시 발송 모듈 단가 캐시 무효화
4. 변경 이력 보관 + 고객사 통보(옵션)
